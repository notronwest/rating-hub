Game Stats · Kitchen Arrival — 4 layout options

Real data from this game. Goals: clearer player attribution, length-proportional bars (not just color), and a sensible spot for the "Add to Stat Review" coach action.

Option 1 — Ranked single list

All four players in one ordered list per role, sorted worst-to-best so the player who needs review surfaces. Bars share a single 0–100% scale so length = magnitude. Team affiliation lives on the avatar color and a small chip.

Option 3 — Player stat cards

Each player gets a card with the % as the dominant element (fast scan, no length comparison needed). Thin proportional ribbon under the % gives the eye a length signal too. "+ Add to review" button always visible at the bottom of each card — no hover-to-reveal.

Option 4 — Twin-bar table

Both roles in one row per player. Each role gets its own bar in its own column (full column width = 100% scale), so length is comparable across players AND roles. Action column on the right has separate "+ S" / "+ R" buttons — coach picks exactly which role to review, no hovering needed.

Trade-offs at a glance
Option 1 · Ranked list — best for "who needs work?" — sorts the worst to the top automatically. Loses the team-mirrored court metaphor.
Option 2 · Mirrored same-scale — keeps the team grouping but fixes the proportional bug. Each role still needs its own block.
Option 3 · Stat cards — % is the hero element, no bar-length comparison required. Action button is always visible. Takes more vertical space.
Option 4 · Twin-bar table — densest. Both roles for a player visible in one row, easy A/B compare. Buttons are explicit per role.
